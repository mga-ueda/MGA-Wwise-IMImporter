# MGA Wwise Importer

Nuendo の tracklist XML と Wave を読み、波形プレビューと将来の分割出力の計画表示を行う Windows 向けツール（開発中）。実際の WAV 分割／Wwise 取り込みは未実装です。

## 使い方

`.wav` または同名ペアの `.xml` をドロップします（波形には `.wav` が必須）。同名 `.xml` があれば小節・テンポ・拍子・マーカー・リージョンを重ねます。起動時は `MgaWwiseImporter.ini` の `AutoLoadWavePath` があれば自動読み込みします。

| 操作 | 内容 |
|------|------|
| Space | 再生／一時停止。シアンのシークバーが追従し、軌跡として残光を残す（停止時は消す） |
| 波形上のマウス | 白い縦線でマウス位置を表示 |
| 波形クリック／ドラッグ | 再生位置を移動 |
| ドロップ | `.wav`（＋同名 `.xml`）を読み込み、プレビューを更新 |

### プレビュー表示

上部にラベル行（小節番号／テンポ／拍子／マーカー／サイクル）、下部に波形です。

- **小節番号** … 波形先頭基準の相対番号（1 始まり）。アウフタクト時は先頭半端小節が 1
- **テンポ・拍子** … 内部では各位置の値を保持し、表示は値が変わった位置だけ
- **マーカー／サイクル** … Nuendo マーカーを行表示。サイクルは範囲塗り
- **リージョン着色** … テンポ変化・拍子変化・サイクル In/Out などで分割。`-R` で終わるサイクル内は除外色（出力計画外）
- **出力計画ラベル** … 連続する除外以外の範囲を `元名_1.wav` … として波形上に表示（書き出し自体は未実装）
- **TimeReference** … iXML（`BWF_TIME_REFERENCE_*`）のみ。無い／0 のときは波形先頭＝PPQ 0 扱い
- **読み込み演出** … ラベル → 波形ワイプ → 小節 → マーカー → サイクル → キャプションの順で重ね表示

波形範囲外のマーカー／サイクルは描画せず、ログの「波形範囲外（無視）」に出します。

## ソース構成

| パス | 役割 |
|------|------|
| `Program.cs` | エントリポイント |
| `UI/` | フォーム・波形ビュー・色定義・ウィンドウ／INI 設定 |
| `Processing/` | ドロップ処理（Wave／XML → プレビュー＋ログ） |
| `Nuendo/` | tracklist XML・テンポ／拍子マップ・小節境界 |
| `Wave/` | WAV／iXML・ピーク・再生・小節／リージョン／出力計画 |

名前空間はフォルダに合わせます（例: `MgaWwiseImporter.UI`、`MgaWwiseImporter.Wave`）。

UI はダーク基調（`UiColors`）。タイトルバーとログまわりもそれに寄せています。

## 開発メモ

- `test/` はローカル検証用でリポジトリには含めません（`.gitignore`）

### `MgaWwiseImporter.ini`（`[Developer]`）

exe と同じフォルダに置きます（無ければ起動時に既定値で作成／不足キーを追記）。`[Window]`（位置・サイズ）とは独立です。

| キー | 意味 | 既定 |
|------|------|------|
| `AutoLoadWavePath` | 起動時の自動読み込みパス（空ならスキップ）。相対パス可 | リポジトリの `test\a.wav`（無ければ空） |
| `TopMost` | ウィンドウを最前面に固定（`1`/`0`） | `0` |

```ini
[Developer]
AutoLoadWavePath=D:\GitHub\MGA-Wwise-Importer\test\a.wav
TopMost=0
```
